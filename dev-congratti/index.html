  • text-me-smth-nice

    View project here

    For this project, I wanted to challenge myself a little and create a simple messaging service that will send out a nice text to someone, every once in a while.

    I started with basically no knowledge, and received a lot of help from asking questions on StackOverflow. The problem I struggled with most was figuring out the tech stack, and what suited my use case best.

    This is how it currently runs: Google App Engine hosts the Python & Flask API route → Heroku hosts a Python script that runs on a schedule to periodically call the Twilio messaging API → and the user makes a call to the API when they submit a form on my website with their phone number.

    There are more features I'd like to build, so I will work on polishing the app a bit more. Overall, I had a lot of fun building it, and I've received positive feedback on it so far.

  • Spotify Recently Added

    View project here

    Before I was introduced to Spotify, I listened to all my music on the iTunes app. One feature I enjoyed was the 'Recently Added' playlist, that iTunes auto-generates based on the top 20 tracks you most recently added to your music library.

    Spotify doesn't offer this feature, but I was excited to find that they have a really well-documented API for developers to use.

    You can check out the app at the link above. You can select the number of your most recently added songs you want to be made into a playlist, and it will ask to authenticate your Spotify account.

    The biggest challenge I had during this app was figuring out how the Spotify authentication worked. It took me a while to figure out, but finally I realized that the way the auth process worked was this: Log in to Spotify with the developer key → Redirect back to the app with a fresh code → Fetch that code from the url use it to make requests to the API.
