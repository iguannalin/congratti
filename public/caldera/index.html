Project

CALDERA is an open-source cybersecurity attack simulation software built in Python.

Opportunity

Balancing act: design a dashboard-like interface that gives the user a sense of trust and control over the simulation tool; develop an app that meets the team’s technical requirements.

UI goals

"Dashboard" display that fits the needs of the cybersecurity researcher using it. It takes inspiration from car dashboard, command centers, as well as digital dashboard interfaces like music software. Like a big tool box, it will provide guidance for the amateur user, and all the visual feedback an expert user seeks.

Challenges

Without prior knowledge of cybersecurity software, there was a learning curve for me to get up to speed with the team, to become familiar with the terms, and relationships between features of CALDERA.

The operations page, one of the most heavily-used features of CALDERA.

Trade-offs

To meet the team’s criteria that the app must run locally, offline, and have a minimal size, I made the decision to not explore frameworks like React, or Vue. Instead, I chose to use a lighter Javascript library for dynamic templating, along with HTML and CSS.

Solution

Rewrite old UI from Jinja & JQuery, to HTML, CSS and Alpine.js. The new user interface features a sidebar, "browser" tabs, and clearer information hierarchy.

Create a standard for future CALDERA UI developments.

Retrospect

Deliver a thorough brand design document in a way that explains design to developers and future designers, and introduce Tailwind CSS earlier.
