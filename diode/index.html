Project

Visualize a lot of information and condense it into a dashboard interface for clients to analyze their component orders.

This bill of materials (BOM) explorer interface borrows heavily from Google Flights’s interface, which is an excellent example of an online tool that is familiar to most people, and an insightful dashboard for people looking to make a decision based on time and money soon.

Dashboard Interface

What is the key piece of information that users need from this page?

Time and cost are both displayed at the top of the page, and are clickable buttons that will update the selected offers in the BOM list.

Like a dashboard, all the call-to-action items are highlighted buttons. Below the time and cost filters are more refined filters. I re-organized the original quantity and US/Global filters, and added two more filters for lead time and price per unit.

They are all filters that operate on the BOM list, so they are styled as buttons and prioritized by placement. The next section shows the totals of what is currently selected in the BOM list.

Component Offers

In place of the table layout, I displayed each component as a list item, and all available offers for each component within an expandable drawer for accessibility and for visual clarity.

They can be sorted, as before, using the row of different sorting methods above the table.

Quick details about the chosen offer can be seen from the collapsed list item, and further details about offers, and the option to choose different offers, can be found from expanding the list item.

Each item in the list shows the following information:

  • Indexing count
  • Designators, with component image popover
  • Quantity x Description
  • MPN code
  • Lead time
  • Price
  • Number of offers available
